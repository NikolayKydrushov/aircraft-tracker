## Задание
В этом проекте вам предстоит написать интеграцию с внешним сервисом.

__Интеграция — объединение данных, находящихся в различных источниках, и предоставление данных пользователям в унифицированном виде.__

Интеграция с внешним сервисом — это довольно распространенная задача для Python-разработчиков, которую можно встретить на платформах с фриланс-проектами.
В курсовой мы будем работать с API.

__API — это механизм, который позволяет различным программам взаимодействовать друг с другом, используя набор определенных правил.__

Чтобы интегрировать внешние сервисы через API, вам понадобится библиотека, которая помогает в этом. Одним из самых простых вариантов является библиотека requests.

__Важно запомнить, что отправка запроса с помощью requests возвращает объект класса Response, который позволяет получить содержимое ответа, а также убедиться, что запрос отработал без ошибок, с помощью кода статуса ответа.__


## Задание
В этой работе вам нужно будет реализовать программу, которая будет собирать данные о самолетах в воздушных пространствах тех стран, которые вы выберете.
Чтобы получить географические координаты стран, будем использовать API [nominatim.openstreetmap.org](https://nominatim.openstreetmap.org/ui/search.html), а чтобы получить информацию о самолетах — API [opensky-network.org](https://opensky-network.org/).

__Код класса, реализующего получение такой информации вы можете посмотреть [по ссылке](https://drive.google.com/file/d/1V6unacC5UtWbSffO6Bu4mD9z64ZrzW9T/view?usp=sharing)
Пример ответа от [nominatim.openstreetmap.org](https://nominatim.openstreetmap.org/ui/search.html) можно посмотреть [по ссылке](https://drive.google.com/file/d/1vMgNJG58oapTYSe5J5pbcrLPZdUsQOEr/view)__

В ответе нас интересует параметр boundingbox — координаты южной, северной, западной и восточной точки страны.

## Требования к реализации

### Шаг 1: создание абстрактного класса для работы с API
  * Создать абстрактный класс для работы с API сервисов [nominatim.openstreetmap.org](https://nominatim.openstreetmap.org/ui/search.html) и [opensky-network.org](https://opensky-network.org/).
  * Реализовать класс, наследующийся от абстрактного класса, для работы с платформами [nominatim.openstreetmap.org](https://nominatim.openstreetmap.org/ui/search.html) и [opensky-network.org](https://opensky-network.org/).

__Класс должен уметь подключаться к API и получать географические координаты стран и информацию о самолетах, находящихся в воздушном пространстве этих стран.__


### Шаг 2: создание класса для работы с информацией о самолетах
  * Создать класс для работы с информацией о самолетах.
  * В этом классе самостоятельно определить атрибуты: страна регистрации, позывной, скорость полета, высота полета и так далее (всего не менее четырех атрибутов).
  Пример ответа с подробным объяснением значения каждого атрибута можно посмотреть тут: [opensky-network.response.json](https://drive.google.com/file/d/17Eco0XUuegw_amvRjhvoSl9coBsvhokr/view).

__Класс должен поддерживать методы сравнения самолетов между собой по скорости и высоте, валидировать данные, которыми инициализируются его атрибуты.__


### Шаг 3: Создание классов для работы с файлами
  * Определить абстрактный класс, который обязывает реализовать методы для добавления информации о самолете в файл, получения данных из файла по указанным критериям и удаления информации о самолетах.
  * Создать класс для сохранения информации о самолетах в JSON-файл. Дополнительно, по желанию, можно реализовать классы для работы с другими форматами: CSV- или Excel-файлом, с TXT-файлом.

__Данный класс выступит в роли основы для коннектора, заменяя который (класс-коннектор), можно использовать в качестве хранилища одну из баз данных или удаленное хранилище со своей специфической системой обращений.
В случае если какие-то из методов выглядят не используемыми для работы с файлами, то не стоит их удалять. Они пригодятся для интеграции к БД. Сделайте заглушку в коде.__


### Шаг 4: создание функции для взаимодействия с пользователем
Создать функцию для взаимодействия с пользователем. Функция должна взаимодействовать с пользователем через консоль.

**Возможности этой функции должны быть следующими:**
  * Ввести название страны для запроса информации о самолетах из [opensky-network.org](https://opensky-network.org/).
  * Получить топ N самолетов по высоте полета (N запрашивать у пользователя).
  * Получить самолеты по стране их регистрации.

Помимо этого функционала, можно придумать дополнительные возможности, которые покажутся удобными.


### Шаг 5: объединение компонентов
  * Объединить все классы и функции в единую программу. 
  * Покрыть описанный функционал тестами.



## Требования к реализации в парадигме ООП
  * Абстрактный класс и классы для работы с API должны быть реализованы в соответствии с принципом наследования. 
  * Класс для работы с самолетами должен быть реализован в соответствии с принципом инкапсуляции и поддерживать методы сравнения самолетов между собой по скорости и высоте полета. 
  * Классы и другие сущности в проекте должны удовлетворять минимум первым двум требованиям принципов SOLID.

## Выходные данные
  * Информация о самолетах, полученная с разных платформ, сохраненная в JSON-файл. 
  * Отфильтрованные и отсортированные данные о самолетах, выводимые пользователю через консоль.
